## 简介

AI Hub 是一个让 Scratch/TurboWarp 项目接入 AI 能力的扩展插件。

**支持的功能：**
- 在作品中创建多个 AI NPC 角色
- 让角色与玩家进行自然语言对话
- 设置游戏背景和玩家身份
- 角色记得对话历史
- 支持同步/异步两种对话模式

**支持的 AI 服务商：**

| AI | 说明 | 模型 |
|----|------|------|
| DeepSeek | 聊天对话，便宜推荐 | deepseek-chat, deepseek-coder |
| Qwen | 图片识别 + 聊天 | qwen-vl-plus, qwen-max, qwen-turbo |
| Pollinations | 图片生成（免费，无 API Key，无 CORS） | flux, turbo |

## 安装方法

### 导入本地文件

1. 打开 [TurboWarp 编辑器](https://turbowarp.org/editor)
2. 点击左下角「添加扩展」
3. 选择「导入插件」
4. 浏览到本文件夹，选择 `ai-hub.js`
5. 完成！

## AI 服务配置

### DeepSeek（聊天对话，推荐）

1. **获取 API Key**
   - 访问 [DeepSeek Platform](https://platform.deepseek.com/)
   - 注册账号
   - 进入 API Keys 页面创建密钥

2. **在扩展中配置**
   ```
   [设置AI服务为 DeepSeek]
   [设置API密钥为 sk-xxxx...你的密钥]
   [选择AI模型 deepseek-chat]
   ```

### Qwen（图片识别 + 聊天）

1. **获取 API Key**
   - 访问 [阿里云百炼](https://bailian.console.aliyun.com/)
   - 注册/登录
   - 创建 API Key

2. **配置**
   ```
   [设置AI服务为 Qwen]
   [设置API密钥为 sk-xxxx...]
   [选择AI模型 qwen-vl-plus]
   ```

### Pollinations（图片生成，强烈推荐）

**完全免费，不需要 API Key！**

1. **不需要注册**，直接用
2. **配置**
   ```
   [设置AI服务为 Pollinations]
   [选择AI模型 flux]  （可选：flux / turbo）
   ```

> **提示**：Pollinations.ai 是**完全免费**的图片生成服务，
> - 不需要 API Key
> - 不需要注册账号
> - 没有 CORS 问题
> - 直接返回图片 URL
> 
> **强烈推荐用于图片生成！**

## 积木说明

### 配置类积木

| 积木 | 说明 |
|------|------|
| `设置AI服务为 [xxx]` | 选择 AI 服务商 |
| `设置API密钥为 [密钥]` | 输入 API Key |
| `选择AI模型 [模型名]` | 选择具体模型 |
| `设置聊天语言为 [语言]` | 设置回复语言，如「中文」「English」 |

### 游戏背景积木

| 积木 | 说明 |
|------|------|
| `设置游戏背景为 [描述]` | 设置游戏世界观，所有角色共享 |
| `设置玩家信息：名字[N] 身份[I] 背景[B]` | 设置玩家角色信息 |

### 角色管理积木

| 积木 | 说明 |
|------|------|
| `创建AI角色：名字[N] 个性[P] 风格[S] 任务[T]` | 创建新的 AI 角色 |
| `设置角色 [N] 的背景为 [背景]` | 设置角色专属背景 |
| `删除AI角色 [N]` | 删除角色及其对话历史 |
| `清空角色 [N] 的对话历史` | 清空对话记录 |

### 对话积木

| 积木 | 类型 | 说明 |
|------|------|------|
| `对话：角色 [N] 消息 [M]` | 报告器 | 同步对话，等待 AI 响应后返回结果 |
| `发送消息：角色[N] 消息[M] 事件ID[E]` | 命令 | 异步发送，不等待响应 |
| `当收到AI响应` | 帽子 | 异步响应到达时触发 |
| `最后的AI响应` | 报告器 | 获取最近一次 AI 回复 |
| `最后的响应角色` | 报告器 | 获取回复的角色名 |
| `最后的事件ID` | 报告器 | 获取响应的事件 ID |

### 图片积木

| 积木 | 类型 | 说明 | 所需服务 |
|------|------|------|----------|
| `生成图片：提示词 [P]` | 报告器 | 生成图片，返回图片 URL | **Pollinations** 或 Qwen |
| `识别图片：[URL] 问题 [Q]` | 报告器 | 让 AI 描述/分析图片内容 | **Qwen**（始终） |
| `生成角色：提示词 [P]` | 命令 | 生成图片并添加到当前角色的造型列表 | **Pollinations** 或 Qwen |
| `生成背景：提示词 [P]` | 命令 | 生成图片并添加到舞台的背景列表 | **Pollinations** 或 Qwen |

> **推荐**：图片生成使用 **Pollinations**，完全免费、无需 API Key、没有 CORS 问题！
>
> **注意**：`生成角色` 和 `生成背景` 会自动将图片导入到 Scratch 的造型/背景栏，并自动切换到新添加的造型/背景。
>
> **注意**：`识别图片` 始终使用 Qwen 服务，与当前选择的 provider 无关。

## 使用示例

### 示例：简单的 AI 问答

```
当绿旗被点击
设置AI服务为 [DeepSeek v]
设置API密钥为 [sk-xxxx... v]
选择AI模型 [deepseek-chat v]

创建AI角色：名字 [小助手 v]，个性 [乐于助人 v]，风格 [亲切友好 v]，任务 [回答问题 v]

询问 [你想问什么？] 并等待
对话：角色 [小助手 v]，消息 (询问的答案)
说出 (对话的结果)
```

### 示例：异步对话（不卡顿）

```
当绿旗被点击
设置AI服务为 [DeepSeek v]
设置API密钥为 [sk-xxxx... v]

创建AI角色：名字 [村长 v]，个性 [睿智的老者 v]，风格 [说话带有古风 v]，任务 [引导玩家 v]
设置游戏背景为 [这是一个古老的村庄 v]

询问 [你想和村长说什么？] 并等待
发送消息：角色 [村长 v]，消息 (询问的答案)，事件ID [1]
```

```
当收到AI响应
说出 (连接 [村长说：] 和 (最后的AI响应))
```

### 示例：生成 AI 角色造型（推荐 Pollinations）

```
当绿旗被点击
设置AI服务为 [Pollinations v]
选择AI模型 [flux v]

生成角色：提示词 [一只可爱的橙色小猫，像素风格，透明背景]
```

### 示例：生成舞台背景（推荐 Pollinations）

```
当绿旗被点击
设置AI服务为 [Pollinations v]
选择AI模型 [flux v]

生成背景：提示词 [星空下的魔法森林，梦幻风格]
```

### 示例：识别图片（始终用 Qwen）

```
当绿旗被点击
设置AI服务为 [Qwen v]
设置API密钥为 [sk-xxxx... v]
选择AI模型 [qwen-vl-plus v]

说 (识别图片：[https://example.com/cat.jpg] 问题 [描述这张图片])
```

## 常见问题

### Q1：提示「API 密钥无效」

**解决**：
1. 检查密钥是否正确复制
2. 确认账户是否有余额
3. 检查密钥是否过期

### Q2：提示「请求失败」

**解决**：
1. 检查网络连接
2. 确认 AI 服务是否可用
3. 稍后重试

### Q3：对话历史会丢失吗？

**是的**，刷新页面后对话历史会清空。
